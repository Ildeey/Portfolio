## 3. ОПИСАНИЕ ПОТОКОВ ДАННЫХ

### 3.1 Основные потоки данных

#### Поток 1: Добавление проекта в галерею
```
Пользователь заполняет форму (галерея)
    ↓
Отправка данных:
  - projectName: string (название проекта)
  - projectImage: URL (ссылка на изображение)
  - projectDescription: string (описание, опционально)
    ↓
JavaScript обработка (script.js)
    ↓
Проверка валидности данных
    ↓
Добавление в DOM (новая карточка в gallery-grid)
    ↓
Сохранение в localStorage (если реализовано)
    ↓
Обновление счётчика проектов
```

**Данные формы:**
```json
{
  "projectName": "Название проекта",
  "projectImage": "https://example.com/image.jpg",
  "projectDescription": "Описание работы (опционально)"
}
```

#### Поток 2: Открытие модального окна проекта
```
Клик на карточку галереи
    ↓
JavaScript обработка (event listener на .gallery-card)
    ↓
Получение данных проекта (из data-project-id)
    ↓
Заполнение модального окна:
  - modalImage: src (изображение)
  - modalTitle: текст (название)
  - modalDescription: текст (описание)
    ↓
Отобразить модальное окно (#imageModal)
    ↓
Возможность навигации (Пред. / След.)
```

#### Поток 3: Счётчик визитов
```
Загрузка страницы
    ↓
JavaScript код измеряет время сеанса
    ↓
Формирование бинарного кода визита (01010101...)
    ↓
Отображение счётчика в footer CTA секции
    ↓
id="visit-stats" + id="time-spent"
```

## 6. ВЗАИМОДЕЙСТВИЯ И СОБЫТИЯ

### 6.1 Основные события и обработчики

| События | Обработчик | Действие |
|---------|-----------|---------|
| Клик на `.gallery-card` | JavaScript | Открыть модальное окно с проектом |
| Клик на `.modal-next` | JavaScript | Следующий проект в галерее |
| Клик на `.modal-prev` | JavaScript | Предыдущий проект в галерее |
| Клик на `.modal-close` | JavaScript | Закрыть модальное окно |
| Submit форма `#projectForm` | JavaScript | Добавить проект в галерею |
| Загрузка страницы | JavaScript | Инициализация счётчика визитов |
| Скролл страницы | CSS (возможно) | Анимационные эффекты |

### 6.2 Обработка формы добавления проекта

```
1. Пользователь заполняет форму:
   - projectName (обязательное поле)
   - projectImage (обязательное поле - URL)
   - projectDescription (опционально)

2. Валидация (в JavaScript):
   - Проверка наличия значений
   - Проверка корректности URL изображения
   - Проверка формата

3. При успехе:
   - Создание нового элемента .gallery-card
   - Добавление в .gallery-grid
   - Очистка формы
   - Вывод сообщения об успехе в #formMessage
   - Сохранение в localStorage (если реализовано)

4. При ошибке:
   - Вывод сообщения об ошибке в #formMessage
```

## 8. БЕЗОПАСНОСТЬ И ВАЛИДАЦИЯ

### 8.1 Валидация входных данных

**Форма добавления проекта:**
```javascript
// Обязательные поля
✓ projectName - текстовое поле (required)
✓ projectImage - URL (required, type="url")

// Опциональные поля
○ projectDescription - текстовая область
```

### 8.2 Защита от XSS

- Использование `.textContent` вместо `.innerHTML` где возможно
- Экранирование HTML символов при добавлении динамического контента
- Валидация URL перед добавлением в атрибут src

### 8.3 Безопасность внешних ссылок

```html
target="_blank" rel="noopener noreferrer"
```
- `target="_blank"` - открыть в новой вкладке
- `rel="noopener"` - предотвратить доступ к window.opener
- `rel="noreferrer"` - не отправлять referrer информацию
